# Chaos项目实践与思考

## 1. 项目感想

1. 2020-02-15

到目前为止，图谱的构建工作还在进行。在构建本体的过程中，有一些思考，现在记录下来：

- 关联性是图谱的关键，构建图谱的核心意义就是希望通过图谱挖掘出实物的深层次关联关系。有些关联性是不直接的，不易发觉，通过图谱可以很直观的展现出来。
- 实体节点有属性信息，同时关系也是有属性的。同一种关系，但连接不同的对象就会有差异，这种差异需要通过属性来表示。

## 2. 关于方向的思考

2020-04-26

最近三天参加知识工厂举办的《知识图谱》培训，除了知识上的一点收获，对最近部门知识图谱项目的一些方向上的困惑，也有了一些思考上的帮助，在这里简单整理一下。

Chaos项目的方向思考：

有两个方向：

1. 以业务和数据为核心，辅助开发相应的工具。

- 人员配套还需要完善，需要数据的专家和业务的专家。让一帮做技术的人针对金融领域的需求构建图谱，很容易找不到方向。
- 明确需求，需要足够清除地定义出图谱的落地场景，解决已有的哪些痛点。
- 要和避开和企业库图谱做重复工作，找好差异化的方向，因为金融数据中的两个最核心的点：公司和人都已被企业库做了。容易做的已经被做了，剩下的就是很难做的。难做的点是数据和业务，这些是我们不擅长的。
- 如果数据不够完整，图谱本身还是很难产生世界落地效果。

2. 以构建知识图谱平台和平台智能化为目标方向。

- 以提供平台为核心，弱化具体的业务和数据。
- 类似于华为的知识图谱云平台。
- 基础功能做拆分，每一块都积极需求落地。
